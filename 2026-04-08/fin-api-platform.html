永不停歇的自我颠覆：介绍 Fin API 平台

作者：Eoghan McCabe | 阅读时间：4分钟

上周，我们宣布了 Apex——世界上第一个专门的客户服务 LLM。现在，我们将允许您通过 API 直接访问所有这些功能和我们所有核心模型。

今天我们宣布推出 Fin API 平台。

我们业界领先的垂直客户服务模型——为 Fin 提供动力——现在可供您混合搭配，并以惊人的规模部署，为您创造完美的客户 Agent。在这篇文章中，我将准确解释我们宣布了什么、为什么这么做，以及我认为我们将如何在整个行业中看到更多这样的情况。

新闻

Fin 是一个客户 Agent 平台，目前每周解决超过 200 万个客户问题，并以快速指数级的速度增长。它被各领域最好的品牌所信赖，无论大小，涵盖你能想象的每一个垂直领域。从 Atlassian 和 Riot Games，到更小的新兴热门公司如 Mercury 和 Polymarket。它运行在我们 AI 团队训练的一系列模型上。上周，我们宣布了 Apex，这是世界上第一个专门的客户服务 LLM。在过去 6 个月的生产测试中，它在解决率、延迟、幻觉率和成本方面击败了每一个前沿模型，包括来自 Anthropic 和 OpenAI 的模型。

现在，我们将允许您通过 API 直接访问所有这些功能和我们所有核心模型，合同起价为每年 25 万美元，每个模型子类别的使用费率是业界最低的。

但是为什么？

原因很简单：我们的客户需要它。我们听到来自各方的人想要构建自己的 Agent。所以从今天开始，我们提供三种方式来实现这一点。

首先，对于绝大多数公司来说，他们希望能在 Fin Agent 平台上运营他们的业务。目前我们约有 8000 家公司在使用它。这满足了 99% 客户的需求，让他们能够轻松配置，而无需经历我们创业公司竞争对手那种令人筋疲力尽的咨询合作。它提供了业界最好的解决率，但开箱即用。

其次，我们还为那些想在独特环境中展示 Fin 的人提供了一个方案。为此，他们可以使用 Fin Agent API。在这种模式下，您可以获得 Fin 平台的所有魔力，但不必使用我们的 Messenger（或我们的电子邮件、语音或其他预建渠道），可以以定制方式展示 Agent。

但也有公司希望为他们的业务构建超特定和专门的 Agent。也许他们想构建一个既提供服务又是产品 Agent 的 Agent，让用户与他们的产品互动。在这种情况下，使用 Apex 和我们在更广泛系统中使用的模型集合是最佳和最明显的选择，因为它们正是为此目的训练的——不像通用模型。这是我们今天推出的第三种全新方案。

我们也很兴奋看到新的创业公司构建类似 Fin 的业务，迎合超特定的垂直领域。牙医的 Fin？为什么不呢？汽车经销商的 Fin？当然可以。我们永远不会为这些特定垂直领域构建，但我们希望其他人来做。事实上，如果我们任何直接的创业公司竞争对手想要大幅改进他们的产品并给我们一点分成，我们也很乐意将我们的模型授权给他们。Decagon、Sierra 和其他公司，你们知道在哪里联系我。让我们做朋友吧！:)

即将出现在你附近的 Agent 公司；防御性原因

与许多人不同，我不是 100% 准备否定所有软件。但确实可以说，软件格局即将在我们眼前发生巨大变化。在最近的时期，差异化来自于软件功能，它充当护城河，因为构建起来很困难。但构建软件现在不再那么困难了。

我们已经成功地将工程团队的可衡量生产力提高了一倍以上。我们还创造了极其深入的新产品，这些产品以前是独立业务，现在由单个 Intercom 工程师在一周内构建完成。

展望未来，来自功能和界面的差异化，至少会减少。严肃的软件公司不仅要从功能公司转向 Agent 公司，因为他们过去协助人类完成的工作现在将由 Agent 完成，而且还必须在这些差异化的 AI 上构建这些 Agent。我们确实相信越来越多的价值将累积到模型层，因此，当我们开始用 Agent 业务颠覆软件业务时，我们现在将开始用 AI 业务颠覆 Agent 业务的过程。

这一切会在哪里结束，任何人都可以猜测，但很难想象我们不会在许多其他公司看到这种情况。目前，我们很高兴能够率先推出这个新平台和业务，我们迫不及待想看看人们会构建什么。
